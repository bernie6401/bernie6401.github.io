    Simple PWN 0x16(simple_smallbin) #

    tags: CTF PWN eductf #

    Version: Ubuntu 20.04

    Background #

    Original Code #

    :::spoiler code

    #include <stdio.h>
#include <stdlib.h>

int main()
{
    void *ptrs[7];
    void *smallbin;
    int i;

    for (i = 0; i < 7; i++)
        ptrs[i] = malloc(0x108); // 0x110 chunk size

    smallbin = malloc(0x108);
    malloc(0x18);

    // aim to fill up tcache
    while(i)
        free(ptrs[--i]);
    
    free(smallbin);
    // trigger unsorted bin dispatch
    malloc(0x870);

    return 0;
}

    :::

    Description & Analyze #

    • First things first, the program will call malloc to get 0x108*8(0x110 chunk size)
    • Then free the all chunks When tcache is fill and chunk size > 0x80, it’ll be put into Unsorted bin
    • And now, if we malloc a new space with size equal 0x870 According to the flow chart, when the malloc size over 0x410, it’ll find Unsorted bin first, and now, Unsorted bin has no suitable chunk, thus find large bin further. Unfortunately, it still has no suitable chunk for the user, split the memory from top chunk
