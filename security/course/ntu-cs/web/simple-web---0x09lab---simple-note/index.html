    Exploit - XSS #

    1. We tried to inject <script> tag in title but has nothing to trigger. According to javascript documentation

      It is not uncommon to see innerHTML used to insert text into a web page. There is potential for this to become an attack vector on a site, creating a potential security risk. Although this may look like a cross-site scripting attack, the result is harmless. HTML specifies that a <script> tag inserted with innerHTML should not execute.

      However, there are ways to execute JavaScript without using <script> elements, so there is still a security risk whenever you use innerHTML to set strings over which you have no control. For example:

      const name = "<img src='x' onerror='fetch()'>";
el.innerHTML = name; // shows the alert
